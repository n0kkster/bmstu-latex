# Стиль для оформления библиографии по ГОСТ 7.0.5-2008

## Краткое описание
Данный файл [gost705.bst](https://github.com/n0kkster/bmstu-latex/blob/main/bibliography/gost705.bst) необходим
для оформления элементов списка использованных источников в соответствии с ГОСТ 7.0.5-2008. Данный файл был найден
мною на просторах гитхаба и немного модифицирован для прохождения нормоконтроля в 2025 году. Никаких гарантий на 
полное соответствие ГОСТу и требованиям кафедры, как и в других шаблонах данного репозитория, не предоставляется.

### Личный опыт использования
- На момент окончания пятого семестра представленный в данном репозитории стиль активно использовался мною 
    по ходу сема во всех письменных работах, а именно: лабах, курсаче, нире. Все работы сданы и защищены без
    проблем, нормоконтроль (у Мальцевой Д. Ю.) пройден дважды, для курсача и для нира.

## Как использовать?

В примере рассматривается следующая структура проекта, при желании можно отойти от нее, но необходимо будет поменять
пути в паре мест:
```
.
└── report
    ├── auto
    │   ├── gost705.bst
    │   └── preamble.tex
    ├── img
    │   └── example.png
    ├── report.pdf
    ├── report.tex
    ├── makefile
    └── parts
        ├── 00-summary.tex
        ├── 01-definitions.tex
        ├── 02-introduction.tex
        ├── 03-analytical.tex
        ├── 04-design.tex
        ├── 05-classification.tex
        ├── 06-conclusion.tex
        ├── 07-bibliography.tex
        └── bibliography.bib
```

1. Качаем репо целиком или данную папку отдельно.
2. [Файл с подключением стиля](https://github.com/n0kkster/bmstu-latex/blob/main/bibliography/0<*>-bibliography.tex) помещается в папку 
    `parts` и подключается в `report.tex` аналогично другим частям документа. Содержимое этого файла представлено в данном репозитории.
3. [Файл с описанием источников](https://github.com/n0kkster/bmstu-latex/blob/main/bibliography/bibliography.bib) также помещается в
    папку `parts`, путь к нему указывается внутри `0<*>-bibliography.tex` относительно места запуска компилятора (в нашем случае 
    относительно `makefile`). В репозитории представлены примеры оформления различных типов источников
